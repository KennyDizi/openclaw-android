## 3. UI 렌더링 레이어 구분

| 화면 | 렌더링 | 이유 |
|------|--------|------|
| 스플래시 | **Native** | www.zip 다운로드 전이므로 WebView 사용 불가 |
| 셋업 진행률 (다운로드/추출) | **Native** | 동일 — WebView 아직 없음 |
| 플랫폼 선택 | **WebView** | www.zip 다운로드 완료 후. OTA 업데이트 가능해야 함 |
| 터미널 | **Native** | PTY + libtermux.so. 성능과 키보드 입력 때문에 네이티브 필수. 멀티 세션 탭 바 포함 |
| 대시보드 | **WebView** | 플랫폼별 동적 UI. www.zip으로 업데이트 |
| 설정 | **WebView** | 기능 추가/변경이 잦음. OTA 업데이트 필요 |
| 탭 바 | **WebView** | 탭 구성이 변경될 수 있음 |
| 업데이트 알림/진행 | **WebView** | 설정 화면 내에서 처리 |

**핵심 규칙**: www.zip이 없는 상태에서는 Native UI만 사용. www.zip 다운로드 완료 후부터 WebView UI 활성화.

## 4. 공통 UI 요소

### 4.1 탭 바 (WebView)

```
┌──────────────────────────────────────┐
│  [🖥 Terminal]  [📊 Dashboard]  [⚙ Settings] │
└──────────────────────────────────────┘
```

- 현재 탭 하이라이트 (색상 또는 밑줄)
- Terminal 탭 선택 시: WebView 숨김, TerminalView 표시
- Dashboard/Settings 탭 선택 시: TerminalView 숨김, WebView 표시
- 탭 전환은 JsBridge `showTerminal()` / `showWebView()` 호출

### 4.2 WebView ↔ TerminalView 전환

```
사용자 [Dashboard] 탭 클릭
  → WebView JS: OpenClaw.showTerminal() 또는 window.location = '#/dashboard'
  → JsBridge: activity.showWebView() 또는 activity.showTerminal()
  → Kotlin: visibility 전환 (GONE / VISIBLE)
  → 전환 시간: 즉각 (애니메이션 없음 — 안정성 우선)
```

### 4.3 상태 배지

```
[⚙ Settings •]     ← 빨간 점: 업데이트 사용 가능
[📊 Dashboard !]    ← 경고: 플랫폼 오류
```

## 5. 디자인 원칙

1. **단순함 우선**: 화면당 하나의 주요 액션. 옵션 최소화.
2. **진행률 투명성**: 다운로드/설치 중 항상 현재 단계와 예상 시간 표시.
3. **실패 복구**: 모든 에러에 "다시 시도" 버튼. 사용자가 막히지 않도록.
4. **터미널 중심**: 터미널이 핵심 기능. 대시보드/설정은 보조.
5. **네이티브 품질**: WebView UI지만 네이티브 앱처럼 느껴야 함 — 빠른 전환, 스크롤 관성, 터치 피드백.

## 6. 표기법

스토리보드 문서에서 사용하는 표기법:

```
[버튼]          ← 탭 가능한 버튼
[버튼 •]        ← 알림 배지가 있는 버튼
━━━━━━          ← 프로그레스 바
→               ← 화면 전환
⟳               ← 로딩/스피너
✓               ← 완료
✗               ← 실패
```

| 태그 | 의미 |
|------|------|
| `[Native]` | Kotlin/XML로 렌더링되는 화면 |
| `[WebView]` | www/ HTML/JS로 렌더링되는 화면 |
| `[JsBridge]` | WebView → Kotlin 호출 발생 |
| `[System]` | 시스템 레벨 동작 (FGS, 알림 등) |
