# skill-stocktake

`.agent/skills/*/SKILL.md` 전체를 감사하여 품질, 중복, 신선도를 평가하고 Keep/Improve/Retire/Merge 판정을 내린다.

## 트리거

- 주기적 감사 (1~2개월마다 권장)
- 스킬이 3개 이상 추가/변경된 시점
- 사용자가 명시적으로 요청 시

## 감사 절차

### Phase 1 — 인벤토리

`.agent/skills/*/SKILL.md`를 전수 스캔한다:

1. 각 SKILL.md의 frontmatter 수집 (name, description, version)
2. `git log -1 --format=%ci` 로 최종 수정일 추출
3. CLAUDE.md T3 테이블에 등록 여부 확인

인벤토리 테이블을 출력한다:

| 스킬 | 버전 | 최종 수정 | T3 등록 | 설명 |
|------|------|----------|---------|------|

### Phase 2 — 품질 평가

각 스킬을 아래 체크리스트로 평가한다:

- [ ] 명확한 트리거 조건이 있는가?
- [ ] CLAUDE.md T3 테이블에 등록되어 있는가?
- [ ] 다른 스킬과 중복되는 부분이 있는가?
- [ ] 마지막 수정이 90일 이내인가?
- [ ] 실행 시 검증 단계가 포함되어 있는가?
- [ ] Termux+스크립트 / 앱+스크립트 양 버전을 고려하고 있는가? (해당되는 스킬만)

평가는 체크리스트 + 종합 판단으로 수행한다. 숫자 점수가 아닌 질적 판단.

### Phase 3 — 판정

각 스킬에 대해 판정을 내린다:

| 판정 | 의미 |
|------|------|
| **Keep** | 유용하고 최신 상태. 변경 불필요 |
| **Improve** | 유지할 가치가 있지만 개선 필요. 구체적 개선안 명시 |
| **Retire** | 저품질, 미사용, 또는 다른 수단으로 대체됨. 삭제 대상 |
| **Merge** | 다른 스킬과 겹침. 병합 대상 스킬을 명시 |

판정 근거는 자체 완결적으로 작성한다:
- Retire: (1) 구체적 결함, (2) 대체 수단 명시
- Merge: 병합 대상과 통합할 내용 명시
- Improve: 어떤 섹션을 어떻게 변경할지 명시

판정 요약 테이블을 출력한다:

| 스킬 | 판정 | 근거 |
|------|------|------|

### Phase 4 — 실행

1. 판정 결과를 사용자에게 보여준다
2. 사용자 확인 후 개선 작업 수행:
   - **Retire**: 사용자 확인 후 파일 삭제 + CLAUDE.md T3 테이블에서 제거
   - **Merge**: 사용자 확인 후 내용 병합 + 원본 삭제
   - **Improve**: 사용자 확인 후 개선 실행
3. 변경된 스킬에 대해 CLAUDE.md T3 테이블 동기화 확인

## 실행 방식

메인 세션에서 직접 실행한다 (subagent 불필요). 스킬 수가 20개 미만이면 한 번에 처리 가능.

## 주의 사항

- 삭제/병합은 반드시 사용자 확인 후 수행
- 판정에 스킬 출처(자작/참조)는 영향을 주지 않음 — 동일 기준 적용
- MEMORY.md와 겹치는 내용이 있으면 중복 해소 제안
